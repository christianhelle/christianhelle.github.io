Unit testing has always been a core part of my development workflow. Over the years, I have used many different libraries and frameworks to help me write better tests. If you have been following my blog, you probably know that I am a big fan of the [Atc](https://github.com/atc-net) libraries. In previous posts, I have written about [Atc.Cosmos](/2023/02/atc-cosmos.html) and [Atc.Cosmos.EventStore](/2024/02/atc-cosmos-eventstore-cqrs.html). Today, I want to talk about [Atc.Test](https://github.com/atc-net/atc-test), a library that brings "A Touch of Class" to unit testing in .NET.

[Atc.Test](https://github.com/atc-net/atc-test) is a helper library designed to streamline the authoring of unit tests by integrating **xUnit v3**, **AutoFixture**, **NSubstitute**, and **FluentAssertions**. The primary goal is to reduce the "ceremony" and boilerplate code, making your tests more readable and easier to maintain.

## Getting started with Atc.Test

The library is built on top of the most popular testing tools in the .NET ecosystem. By combining these, it allows you to focus on what you are testing rather than how to set up the test data and mocks.

### Packages

To get started, you need to add the [Atc.Test](https://www.nuget.org/packages/Atc.Test) NuGet package to your test project.

```xml
<PackageReference Include="Atc.Test" Version="1.0.0" />
```

*Note: Since Atc.Test is built for xUnit v3, make sure your project is configured to use the latest version of xUnit.*

## Auto-generating Test Data

One of the most powerful features of [Atc.Test](https://github.com/atc-net/atc-test) is its integration with [AutoFixture](https://github.com/AutoFixture/AutoFixture). Instead of manually creating instances of your classes and their dependencies, you can use the `[AutoNSubstituteData]` attribute.

```csharp
[Theory]
[AutoNSubstituteData]
public void Add_ShouldReturnSumOfTwoNumbers(
    int a, 
    int b, 
    Calculator sut)
{
    // 'a' and 'b' are random integers generated by AutoFixture
    // 'sut' is automatically instantiated
    var result = sut.Add(a, b);
    
    result.Should().Be(a + b);
}
```

In the example above, `a` and `b` are populated with random integers, and the `Calculator` (System Under Test) is automatically instantiated. If `Calculator` had any constructor dependencies (interfaces or abstract classes), they would be automatically mocked using [NSubstitute](https://nsubstitute.github.io/).

## Mocking with NSubstitute

When you need to verify interactions with dependencies, [Atc.Test](https://github.com/atc-net/atc-test) makes it incredibly easy by automatically providing substitutes for any interface in your test parameters.

```csharp
[Theory]
[AutoNSubstituteData]
public void GetValue_ShouldReturnDataFromService(
    IMyService mockService, 
    MyController sut)
{
    // mockService is an NSubstitute substitute
    mockService.GetValue().Returns(42);
    
    var result = sut.Get();
    
    result.Should().Be(42);
}
```

## Parameterized Tests with Inline Data

Sometimes you want to provide specific values for some parameters while letting AutoFixture generate the rest. For this, you can use `[InlineAutoNSubstituteData]`.

```csharp
[Theory]
[InlineAutoNSubstituteData(10, 20)]
[InlineAutoNSubstituteData(5, 5)]
public void Add_ShouldWorkWithSpecificValues(
    int a, 
    int b, 
    Calculator sut)
{
    var result = sut.Add(a, b);
    
    result.Should().Be(a + b);
}
```

## Sharing Instances with [Frozen]

If you need to use the exact same instance of a dependency across multiple parameters (for example, to set up a mock and then have it injected into your SUT), you can use the `[Frozen]` attribute.

```csharp
[Theory]
[AutoNSubstituteData]
public void Handle_ShouldUseFrozenDependency(
    [Frozen] IMyService mockService,
    MyHandler sut)
{
    // mockService is the same instance that is injected into MyHandler
    mockService.DoWork().Returns(true);
    
    var result = sut.Handle();
    
    result.Should().BeTrue();
    mockService.Received(1).DoWork();
}
```

## Customizing Data Generation

You can define global rules for how certain types should be generated by creating a customization class and decorating it with the `[AutoRegister]` attribute. [Atc.Test](https://github.com/atc-net/atc-test) will automatically discover and apply these rules.

```csharp
[AutoRegister]
public class MyCustomization : ICustomization
{
    public void Customize(IFixture fixture)
    {
        fixture.Register(() => new MyComplexType("Fixed Value"));
    }
}
```

## Advanced Data Sources

For more complex scenarios where you need to provide multiple sets of data, including mocks, you can use `[MemberAutoNSubstituteData]`. This works similarly to xUnit's `[MemberData]` but adds the auto-mocking capabilities.

```csharp
public static IEnumerable<object?[]> TestData()
{
    var mock = Substitute.For<IMyService>();
    mock.GetValue().Returns(100);
    yield return new object?[] { mock };
}

[Theory]
[MemberAutoNSubstituteData(nameof(TestData))]
public void MemberData_Example(
    IMyService suppliedMock,
    MyController sut)
{
    // suppliedMock is the instance from TestData()
    // and it is also injected into sut
    var result = sut.Get();
    
    result.Should().Be(100);
}
```

## Helper Extensions

The library also includes several convenience extensions for **FluentAssertions** and **NSubstitute** to further reduce boilerplate.

- **Equivalency Extensions:** Simplifies configuring object comparisons.
- **Substitute Extensions:** Helpers for inspecting arguments and waiting for async calls.
- **Reflection Helpers:** `ObjectExtensions` provides ways to access protected members when absolutely necessary.

## Conclusion

[Atc.Test](https://github.com/atc-net/atc-test) is a fantastic addition to any .NET developer's toolkit. By removing the repetitive setup code, it allows you to write cleaner, more expressive tests that are easier to maintain. If you are already using xUnit, AutoFixture, and NSubstitute, this library is a no-brainer.

You can find the full source code and more examples on the [GitHub repository](https://github.com/atc-net/atc-test).
