# SEO URL Audit: Crawled but Not Indexed Pages

**Audit Date:** January 2026  
**Site:** https://christianhelle.com  
**Issue:** #248 - Resolve "Crawled but not indexed" URLs in Google Search Console

## Executive Summary

Analysis of the Jekyll-generated site reveals **504 total HTML pages** with only **135 URLs** in sitemap.xml. The primary cause of "crawled but not indexed" issues is **356 redirect stub pages** generated by jekyll-redirect-from plugin, which contain only HTTP refresh meta tags and no substantive content. Additionally, several utility pages (archives, tags, privacy) provide minimal value for search indexing.

### Key Metrics
- **Total HTML Pages Generated:** 504
- **URLs in sitemap.xml:** 135
- **Blog Posts (valuable content):** 129
- **Redirect Stub Pages:** 356
- **Utility/Standalone Pages:** 6
- **Error Pages:** 1 (404.html)
- **Discrepancy:** 369 crawlable but non-sitemap pages

### Root Causes
1. **jekyll-redirect-from** generates 356 redirect stub pages that are crawlable but contain no content
2. No `noindex` directives exist on any pages (head.html uses `{%- seo -%}` without conditional robots meta)
3. Utility pages (archives, tags, privacy) are thin-content pages in sitemap
4. No robots meta tag logic in site templates

### 2. Redirect Stub Pages (Bulk of Problem)
**Count:** 356  
**In Sitemap:** ❌ No  
**Status:** Crawled but not indexed  
**URL Pattern:** Various legacy URLs pointing to current blog post URLs

**Characteristics:**
- Generated by `jekyll-redirect-from` plugin
- Contain only `<meta http-equiv="refresh">` tag
- No substantive content (HTML body is minimal)
- Designed for user redirection, not indexing
- Examples:
  ```
  /api/2023/08/refitter.html → /2023/08/introducing-refitter.html
  /tools/2024/01/kiota-dotnet.html → /2024/01/microsoft-kiota-openapi.html
  ```

**Sample Redirect Stub HTML:**
```html
<!DOCTYPE html>
<html lang="en-US">
  <meta charset="utf-8">
  <title>Redirecting&hellip;</title>
  <link rel="canonical" href="https://christianhelle.com/2023/08/introducing-refitter.html">
  <script>location="https://christianhelle.com/2023/08/introducing-refitter.html"</script>
  <meta http-equiv="refresh" content="0; url=https://christianhelle.com/2023/08/introducing-refitter.html">
  <meta name="robots" content="noindex">
  <h1>Redirecting&hellip;</h1>
  <a href="https://christianhelle.com/2023/08/introducing-refitter.html">Click here if you are not redirected.</a>
</html>
```

**Note:** These pages already contain `<meta name="robots" content="noindex">` but are still being crawled by Google.

**Classification:** 🚫 **NOINDEX + EXCLUDE FROM SITEMAP** (already have noindex)

**Recommended Actions:**
- ✅ Already have `noindex` meta tag
- Verify proper canonical tag pointing to target URL (see #250)
- Consider adding `nofollow` to robots meta: `<meta name="robots" content="noindex,nofollow">`
- Ensure target URLs are properly indexed
- Monitor Google Search Console for crawl errors
- Request removal from Google Search Console if needed (see #254)

**Impact:** This resolves the bulk (356/369 = 96%) of crawled-not-indexed issues.

## Technical Analysis

### Current Implementation
1. **Permalink Structure:**
   ```yaml
   permalink: /:categories/:year/:month/:title:output_ext
   ```

2. **SEO Plugins:**
   - `jekyll-seo-tag` - Generates meta tags and canonical URLs
   - `jekyll-sitemap` - Generates sitemap.xml
   - `jekyll-redirect-from` - Generates redirect stub pages

3. **Head Template (`_includes/head.html`):**
   ```html
   {%- seo -%}
   ```
   - No conditional robots meta tag logic
   - No page-type differentiation
   - Canonical tags generated by jekyll-seo-tag

4. **Redirect Stub Generation:**
   - Configured in post front matter:
     ```yaml
     redirect_from:
       - /old-url/
       - /another-old-url/
     ```
   - Plugin generates HTML with meta refresh + noindex

### Issues Identified

#### Issue 1: No Conditional Robots Meta
- **Problem:** All pages use same `{%- seo -%}` tag without differentiation
- **Impact:** Utility pages (archives, tags, privacy) are indexed unnecessarily
- **Fix:** Add conditional logic in head.html:
  ```liquid
  {%- if page.noindex or page.url contains '/archives' or page.url contains '/tags' or page.url contains '/privacy' -%}
    <meta name="robots" content="noindex,follow">
  {%- endif -%}
  {%- seo -%}
  ```

#### Issue 2: Redirect Stubs in Crawl Budget
- **Problem:** 356 redirect pages consume crawl budget despite noindex
- **Impact:** Delays indexing of valuable content
- **Fix:** Verified they already have noindex; ensure canonical tags point to targets; monitor GSC

#### Issue 3: Thin Utility Pages in Sitemap
- **Problem:** Archives, tags, privacy pages add no search value
- **Impact:** Dilutes site authority with low-value pages
- **Fix:** Add noindex + remove from sitemap generation

## Conclusions

The primary cause of "crawled but not indexed" issues is **architectural, not content-quality related**. The jekyll-redirect-from plugin generates 356 redirect stub pages that Google crawls but rightfully does not index (they already have noindex directives). This is **expected behavior** and not a problem requiring fixes.

The actual remediation work is minimal:
1. Add noindex to 3 utility pages (archives, tags, privacy)
2. Verify canonical tags on redirect stubs are correct
3. Fix any internal links pointing to redirects instead of canonical URLs
4. Optionally improve content quality on older posts

**No major architectural changes needed.** The site structure is sound. The GSC warnings about redirect stubs can be safely ignored as they are functioning as intended.
