- **2026-03-20 (Style Analysis):** Analyzed two recent blog posts (clocz, argiope) to extract voice, structure, and conventions.
  - Voice: conversational, practical, humble. Avoid hype and corporate jargon.
  - Structure: 4–6 major sections with implicit transitions. Open with hook + GitHub link.
  - Code blocks: 15–40 lines, use ellipsis (`// ...`) for brevity, minimal inline comments.
  - Front matter: layout, date, author, tags (lowercase). Redirect_from only for URL changes.
  - Conclusion: reflect on experience, end with GitHub link.
  - Post length: ~2000–2500 words (medium-long form).
  - Section rhythm: "How it works" → implementation deep dives → usage & output → distribution → conclusion.
